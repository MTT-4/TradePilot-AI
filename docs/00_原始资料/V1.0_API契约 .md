## 0. 通用约定

- **Base path**：业务 API 在 `/api/*`；公开追踪跳转在 `/t/:slug`；公开表单在 `/api/public/*`；webhook 在 `/api/webhooks/*`。
- **鉴权**：登录后下发会话 Cookie（HttpOnly+Secure）。除公开端点外都需登录。
- **租户上下文**：请求头 `X-Tenant-Id`，服务端**必须校验该用户在此租户有 membership**，否则 403。所有查询按此 `tenant_id` 过滤（硬约束 #1）。
- **RBAC**：每个端点标注最小角色。`viewer` 只读；`sales` 仅自己负责的线索；`operator` 可建内容但对外动作需审批；`admin/owner` 全权 + 审批。
- **错误信封**：`{ "error": { "code": "...", "message": "...", "details": {} } }`。
- **错误码**：`VALIDATION`(400) · `UNAUTHENTICATED`(401) · `FORBIDDEN`(403) · `NOT_FOUND`(404) · `CONFLICT`(409) · `UNPROCESSABLE`(422) · `RATE_LIMITED`(429) · `HITL_REQUIRED`(409) · `LOCAL_MODEL_UNAVAILABLE`(503，隐私任务排队不 fallback) · `INTERNAL`(500)。
- **分页**：`?cursor=&limit=`（默认 20，上限 100）→ `{ items, nextCursor }`。
- **异步**：耗时操作（上传解析、向量化、生成站/内容包、翻译）返回 `202 { jobId }`，前端轮询 `GET /api/jobs/:id`。
- **幂等**：webhook 与会重复的写操作要求 `Idempotency-Key` 头；命中已处理返回原结果（不重复造数据）。
- **前期运行环境**：单机部署在创始人 Mac M5 128G——应用 + Postgres(pgvector) + Redis + MinIO + llama.cpp(Qwen/bge-m3) 同机。模型网关用配置区分本地端点。适合开发与首个低并发试点；扩容/7×24 时迁云或独立服务器（仅改配置）。

## 1. 鉴权 / 账户

| 端点 | 角色 | 请求 → 响应 |
|---|---|---|
| `POST /api/auth/register` | 公开 | `{email,password,name}` → `{userId}`（触发 2FA 绑定引导） |
| `POST /api/auth/login` | 公开 | `{email,password}` → `{status:"ok"}` 或 `{status:"2fa_required",challengeId}` |
| `POST /api/auth/2fa/verify` | 公开 | `{challengeId,code}` → 会话建立 |
| `POST /api/auth/logout` | 登录 | → `204` |
| `GET /api/me` | 登录 | → `{user, memberships:[{tenantId,role,tenantName}], currentTenant}` |
| `POST /api/tenants` | 登录 | `{name,defaultLocale}` → `{tenantId}`（创建者成为 owner） |
| `GET /api/members` | admin+ | → 成员列表 + 角色 |
| `POST /api/members` | admin+ | `{email,role}` → 邀请 |
| `PATCH /api/members/:id` | owner/admin | `{role|status}` → 更新（写 audit_log） |
| `GET /api/usage` | admin+ | → `{creditsBalance, recent:[model_invocations 汇总]}` |

## 2. 知识库

| 端点 | 角色 | 请求 → 响应 |
|---|---|---|
| `POST /api/kb/documents` | operator+ | multipart 文件 或 `{sourceType:"url",url}` + `{sensitivity}` → `202 {documentId,jobId}` |
| `GET /api/kb/documents` | viewer+ | → 列表（含 `status`：uploaded→parsing→chunking→embedding→ready/failed） |
| `GET /api/kb/documents/:id` | viewer+ | → 文档详情 + chunk 数 + 待审核数 |
| `GET /api/kb/reviews?status=pending` | operator+ | → 知识卡片列表（question/answer/source_citation/sensitivity） |
| `POST /api/kb/reviews/:id` | operator+ | `{action:"approve"|"correct", correctedText?, sensitivity?}` → 更新 |
| `POST /api/kb/search` | 内部服务 | `{query,filters,allowInternalOnly}` → chunks（**按 tenant 隔离；allowInternalOnly=true 仅本地 Qwen 路径可调**） |

## 3. 建站智能体

| 端点 | 角色 | 请求 → 响应 |
|---|---|---|
| `POST /api/sites/generate` | operator+ | `{brief:{market,product,locales,style,cta}}` → `202 {siteId,jobId}`（仅检索 public 知识） |
| `GET /api/sites` | viewer+ | → 站点列表（status：draft/published/offline） |
| `GET /api/sites/:id` | viewer+ | → project + pages + locales + 当前版本 + SEO/GEO/OG 状态 |
| `POST /api/sites/:id/chat` | operator+ | `{message}` → 修改建议/新版本草稿 |
| `POST /api/sites/:id/autofill` | operator+ | → AI 按知识库生成的候选内容（产品页/认证/博客），**每条待确认** |
| `POST /api/sites/:id/publish-request` | operator+ | → `{hitlTaskId}`（**不直接上线**，建 hitl_task） |
| `POST /api/sites/:id/rollback` | operator+ | `{versionId}` → 回滚 |

## 7. 首响 / HITL

| 端点 | 角色 | 请求 → 响应 |
|---|---|---|
| `POST /api/replies/draft` | sales+ | `{inquiryId}` → `{replyId,draftText,citations}`（**强制走 local_qwen；本地不可用→503 排队，不 fallback**） |
| `PATCH /api/replies/:id` | sales+ | `{draftText}` → 编辑草稿 |
| `GET /api/hitl?status=pending` | operator+ | → 待审批（type：site_publish/content_publish/reply_send） |
| `POST /api/hitl/:id/approve` | 取决 type（reply→sales+；publish→admin 可配） | → 审批通过**才真正执行对外动作**（上线/发布/发送），写 audit_log |
| `POST /api/hitl/:id/reject` | 同上 | `{reason?}` → 拒绝 |

## 9. 鉴权矩阵（角色 × 关键操作）

| 操作 | owner | admin | operator | sales | viewer |
|---|:-:|:-:|:-:|:-:|:-:|
| 看数据 | ✓ | ✓ | ✓ | 仅自己线索 | ✓ |
| 改成员/权限 | ✓ | ✓ | ✗ | ✗ | ✗ |
| 建内容/站点 | ✓ | ✓ | ✓ | ✗ | ✗ |
| 审批上线/发布 | ✓ | ✓ | 可配 | ✗ | ✗ |
| 起草/发送首响 | ✓ | ✓ | ✓ | ✓(自己) | ✗ |
| 看用量/账单 | ✓ | ✓ | ✗ | ✗ | ✗ |

> 所有"对外动作"端点（publish-request、mark-published、replies 发送）必须经 HITL 审批后生效；无静默对外路径。
