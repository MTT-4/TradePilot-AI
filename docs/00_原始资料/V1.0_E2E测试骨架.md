## 0. 工具与种子

- **UI E2E**：Playwright。**API/集成**：Vitest + supertest（或 Playwright APIRequest）。
- **本地模型**：测试用 mock 一个 OpenAI 兼容端点模拟 Qwen/bge-m3；另设"本地不可用"开关用于故障测试。
- **种子数据（T0.13）** `seed.ts`：
  - 租户 **A=晟海机械**、**B=对照公司**；各 1 个 owner + 1 个 sales。
  - 租户 A：样本知识库（产品手册/报价[仅内部]/认证）、1 个站点项目、1 个内容包（9 平台 items + tracking_links）、2 条样本询盘（表单 1 + 邮件 1）。
  - 平台规则库 9 条。

## 1. 红线测试（必须全绿，否则阻断）

### R1 租户隔离
- **R1.1** 用 B 的会话带 `X-Tenant-Id=A` 调任意读端点 → 403 / 空，且写 audit_log。
- **R1.2** 直接用 B 会话查 A 的 lead/document/site id → NOT_FOUND（不泄露存在性）。
- **R1.3** 向量检索：以 B 身份检索，绝不返回 A 的 chunk（按 tenant 隔离 namespace）。

### R2 隐私路由（不泄密）
- **R2.1** 标 `internal_only` 的报价知识，不出现在任何发往 OpenAI/Google 的 prompt（拦截 mock 出站请求断言）。
- **R2.2** 含邮箱/电话/询盘正文的首响草稿：`replies.model_invocation.route == local_qwen`；mock OpenAI 端点零调用。
- **R2.3** **本地引擎不可用**时，首响请求 → 503 `LOCAL_MODEL_UNAVAILABLE` 并排队；**断言绝不 fallback 到 openai/google**。
- **R2.4** `model_invocations.contains_pii=true` 的记录，route 必 ∈ {local_qwen, local_bge}。
- **R2.5** **分类阶段零第三方调用**：对含/疑似隐私输入，DataClassifier 判定过程中 mock 的 OpenAI/Google 端点调用数 = 0（分类只走本地规则/本地模型，防分类即泄密）。

### R3 追踪不断链（服务端权威）
- **R3.1** 点击 `/t/:slug` → 写 click_event（campaign/content/platform 来自 tracking_link）。
- **R3.2** 篡改 URL 上的 UTM → 归因仍以服务端 tracking_link/campaign_id 为准。
- **R3.3** 链路贯通：click → 表单提交 → lead → inquiry → opportunity，`lead.source_content_item_id/tracking_link_id` 可回溯到 content_item/campaign；dashboard sourceAttribution 计数 +1。

### R4 HITL 闸门
- **R4.1** 未审批时：site 不上线 / content_item.publish_status 不为 published / reply 不发送。
- **R4.2** approve 后才真正对外生效，并写 audit_log（含操作者）。
- **R4.3** viewer 角色看不到、也调不动任何对外动作端点（403）。

### R5 入站邮件幂等 / 去重
- **R5.1** 同一封邮件 webhook 重投（同 Idempotency-Key 或同 dedupe_hash）→ 只产生 1 条 inbound_email / lead。
- **R5.2** 垃圾邮件 → status=spam，不进线索池。

## 2. 功能链路测试（里程碑级）

- **F-M1 知识库**：上传 PDF → job 走 parsing→chunking→embedding→ready → 可检索 → 审核可标已核准/敏感级。
- **F-M2 建站**：brief → 生成 EN/AR/RU 站；ar 为 rtl；URL 带语种后缀；publish-request 建 hitl。
- **F-M3 内容包**：选题 → 9 平台 items；视频平台 media_type=video_script（脚本/分镜/封面，无成片）；每 item 有 tracking_link；导出可用。
- **F-M4 CRM**：表单 + 邮件两来源 → 去重 → 线索池含来源归因 → 商机管道改 stage。
- **F-M5 首响/看板**：inquiry → 本地 Qwen 起草 → HITL 发送 → dashboard 显示来源。

## 3. 完整闭环验收剧本（对应验收报告第 6 节 · 18 步）

按"样本公司空压机中东获客"跑通，全部断言通过才算 V1.0 闭环通过：

1. 初始化租户 A/B → R1 全绿。
2. A 上传产品 PDF/Excel → 生成 document/chunks/job。
3. 审核知识卡片 → 可标已核准/仅内部/可公开。
4. 向量化入库 → 按 tenant + namespace 隔离。
5. 生成中东落地页 → 仅引用 public 知识。
6. 预览 EN/AR/RU → 阿语 RTL、表单可用。
7. 确认上线 → 产生 HITL 记录。
8. 生成 9 平台内容包 → 图文可用；视频平台有脚本/分镜/封面。
9. 生成追踪链接 → 每 content_item 唯一 tracking_link。
10. 手动发布并标记已发 → 状态更新 + 操作者，绑 content_item_id。
11. 点击追踪链接 → click_event 记录 campaign/content/platform。
12. 提交表单询盘 → lead/inquiry 生成。
13. 来源归因 → 可追溯 platform/content/tracking_link。
14. 自动去重进管道 → 联系人/商机不重复。
15. 生成首响草稿 → 仅本地 Qwen。
16. 审批并发送首响 → HITL 记录完整。
17. 看 dashboard → 显示该询盘来自 X 平台 Y 内容。
18. 工作台下钻 → 点数字/节点进对应详情页。
