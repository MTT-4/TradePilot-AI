# AI 外贸营销智能体 · V1.0 MVP 产品设计书

> **说明：** 本文是锁定 7 项关键决策后的 V1.0 可落地设计，作为开发依据。范围＝MVP 闭环，超出部分见“V1.5 / V2 预留”。三角色讨论与决策依据见配套《五大模块需求深挖讨论纪要》。
> **版本冻结：** 本版按最新决策执行：**轻量短视频成片 / TTS / 自动字幕 → V1.5；复杂多租户代运营后台 → M7；除这两类外，V1.0 主闭环功能必须全做。**

---

## 1. V1.0 决策锁定

| # | 议题 | 决策 |
|---|---|---|
| 1 | 首发模型 | **OpenAI**（公开营销内容生成）＋ **Google 翻译**（营销内容翻译）＋ **本地 Qwen**（涉及客户隐私数据的任务）；多模型路由 V1 即按“数据敏感度”分流 |
| 2 | 邮件归集 | 平台为每个用户分配**专属系统邮箱**接收询盘 |
| 3 | 验证方式 | 暂无外部试点，先**内部自跑**（用真实样本产品/公司）；尽快补真实客户 |
| 4 | 域名/网站 | V1 用**平台二级域名**；网站**响应式**；**每语种独立 URL（带语种后缀）** ＋ hreflang |
| 5 | SEO/GEO | SEO＋GEO **优化均进 V1**；GEO 可见度**监测面板 → V1.5** |
| 6 | 默认语种 | **英、阿、俄、法、德、葡** 共 6 种（阿语含 RTL 适配） |
| 7 | 验收标准 | 闭环跑通线（样本公司版）＋ 愿继续用/付费的软指标 |
| 8 | 视频范围 | V1.0 覆盖视频平台的**脚本、分镜、封面、文案、标签、规格校验、追踪链接、发布清单**；不做短视频成片、TTS、自动字幕 |
| 9 | 多租户范围 | V1.0 必做基础租户隔离；复杂代运营跨租户后台放 M7 |

## 2. V1.0 闭环全景

> 知识库（导入→审核→已核准）→ AI 建站智能体（响应式多语言落地页／每语种独立 URL／内置 SEO+GEO／询盘表单／预埋追踪链接）→ AI 设计（图文内容包＋视频平台脚本/分镜/封面＋每份预埋追踪链接＋发布清单）→ **人工发布到社媒平台**（平台外唯一手动动作）→ 访客点链接进落地页 → 表单/邮件询盘 → 统一线索池（去重＋UTM 来源归因）→ 最小 CRM（商机管道＋跟进提醒）→ AI 多语言首响草稿（人工一键发）＋规则打分 → 看板回看来源 → 反哺下一轮选题

**贯穿主线：追踪链接** 把建站、设计、CRM、看板串成可量化闭环，须最先打通。  
**HITL 说明：** 平台外唯一手动动作是社媒发布；平台内所有对外动作（网站上线、内容发布确认、客户回复）默认进入 HITL 人工确认队列。

---

## 3. 核心设计原则

1. **追踪链接是隐形主线**——所有对外内容预埋专属链接，是闭环可量化的根基。
2. **HITL（人工确认）**——对外动作（网站上线、内容发布、回客户）默认人工一键确认。
3. **抗幻觉护栏**——强制先检索知识库再生成，无依据则提示补充而非编造。
4. **可配置**——平台规则、模板、语种、打分规则皆做成配置，不写死。
5. **最小自研但闭环完整**——CRM、建站、内容包均取最小可用形态，但 V1.0 主闭环必须跑通；短视频成片、TTS、自动字幕进入 V1.5。
6. **隐私优先**——第三方模型调用前必须经过数据敏感度分类；客户隐私与内部资料不得出境。

## 4. 模型与翻译层

**架构：** 上层 Agent 调用统一接口（`LLM.invoke` / `Translate` / `Embed`），底层可插拔。V1 即接入三方——OpenAI、Google 翻译、**本地 Qwen**——并启用**按数据敏感度路由**：公开营销内容走 OpenAI，公开营销翻译走 Google，涉客户隐私或内部资料的任务走本地 Qwen。抽象层为 V2 接入更多模型铺路。

**模型分流规则（V1 即生效）：**

| 任务类型 | 是否含客户隐私/内部资料 | 走哪个 |
|---|---|---|
| 建站文案、社媒图文、博客等公开营销内容 | 否 | OpenAI 生成 |
| 上述内容的多语言翻译 | 否 | Google 翻译（术语表锁定） |
| 询盘内容理解、AI 首响草稿、跟进文本、含个资的处理 | 是 | **本地 Qwen**（多语言直出，不经第三方） |
| 标记“仅内部”的知识块、报价、合同、未公开案例 | 是 | **本地 Qwen**；不得被 OpenAI/Google 使用 |

**强制隐私路由：** 第三方模型调用前必须经过 `DataClassifier`。只要检测到姓名、邮箱、电话、公司联系人、客户询盘正文、报价、合同、内部案例、未公开资料，或知识块标记为“仅内部”，一律强制走本地 Qwen。**本地 Qwen 不可用时不得 fallback 到 OpenAI/Google，只能提示“隐私模型暂不可用，请稍后重试或人工处理”。** 每次路由必须写审计日志：`tenant_id`、`user_id`、`task_type`、`sensitivity`、`provider`、`reason`、`timestamp`。

**本地模型选型（已确定）：**
- **向量模型：bge-m3**（多语言、覆盖 6 语种、支持语义＋关键词混合检索；本地嵌入，知识库向量化与存储均在本地）。
- **隐私 LLM：Qwen2.5-VL-32B-Instruct（标准 Instruct 版，Q8）**；明确**排除 uncensored / aggressive 变体**（去安全对齐，不适合商用客户沟通）。可与 Qwen3.6-35B-A3B 在真实样本上 A/B 后定稿。

**内容生成流程：**
1. 源语言内容由 **OpenAI** 生成（仅基于可公开知识库检索结果）。
2. 其余语种由 **Google 翻译** 产出，使用**翻译术语表（glossary）锁定**型号、品牌名等专有名词。
3. 标语、口号、CTA 等高营销度文案：由 OpenAI 直接出目标语版本（transcreation）或人工复核，避免机翻生硬。
4. 询盘、首响、客户跟进、内部资料问答：只走本地 Qwen。

**数据合规提示：** OpenAI 与 Google 为境外处理，**仅用于不含客户隐私的公开营销内容**。所有涉及客户个人信息（姓名/邮箱/电话/询盘正文）的处理走**本地 Qwen**，数据留在平台可控环境、不出境、不发第三方。  
**本地 Qwen 部署分阶段：** ① 开发/自测阶段——跑在开发者**个人电脑**（用 llama.cpp 的 `llama-server` 暴露 OpenAI 兼容端点，仅供开发，不扛生产流量），按显存/内存选量化版；② 接入真实客户/正式上线前——迁移到**自有云 GPU 或本地服务器**（须为自己控制的环境，不可换成第三方 AI 接口，否则隐私失效）；③ 客户机房真·私有化 → V2。**关键：Qwen 一律置于标准推理端点（OpenAI 兼容）之后，应用只认端点地址，换环境＝改配置、代码不动。GPU 与运维成本随阶段递增，须计入预算。**

## 5. 五大模块 V1.0 范围

### 5.1 AI 知识库（RAG）
**自动入库流程（上传即用，全自动、异步带进度）：** 拖拽上传（PDF/Word/Excel/网址）→ 自动解析取文本 → 自动切块（保留表格/参数结构）→ **bge-m3 本地向量化** → 存向量库 → 自动打标（产品/市场/语言/来源/敏感级）→ 立即可被检索。向量化与存储均在本地，proprietary 资料不出门。  
**做：** 上述自动入库；手动补问答对；**知识审核页**（确认/修正/标“已核准”，提升准确度、不阻塞可用性）；强制溯源；统一检索接口（bge-m3 语义＋关键词混合检索，按产品/市场/语言/敏感级过滤）；抗幻觉护栏（无依据不编造）；中文为源 ＋ 术语表锁定专有名词。  
**隐私分级：** 资料可标“仅内部”（报价/内部案例等）——仅供本地 Qwen 调用，不随营销内容发给 OpenAI/Google；可公开的产品参数两边皆可用。  
**不做（→V2）：** 图片 OCR、知识图谱、知识更新后自动重生成下游内容（V1 只做影响范围提示）。

### 5.2 AI 建站智能体
**做：** 对话＋半结构化引导（5–6 个必填项）生成落地页/专题页，内容自动取知识库；单次 ≤3 页；生成后可继续对话修改；**双轨管理**＝对话改内容 ＋ 轻量管理台（站点列表/上下线/版本快照回滚/预览/HITL 发布）；内容补充复用生成管线；AI 起草博客（取知识库＋审核）。  
**AI 按知识库自动补全内容（建成后持续）：** 网站建成后，新产品页/参数/认证/博客由 AI 依据知识库自动生成草稿并上线候选，**每条均需人工“确认上线”，且随时可调整/编辑**（AI 自动 ＋ 人工把关，符合 HITL）。  
**网站基线：** **响应式（移动优先）**；平台二级域名；**每语种独立 URL（语种后缀/子路径）＋ hreflang**；**阿拉伯语 RTL 适配**；海外 CDN（注意不要默认拦截 AI 爬虫）。  
**默认内置 SEO＋GEO：** 见第 7 节。  
**不做（→V2）：** 拖拽全站编辑器、复杂 CMS、自定义域名（列为紧随其后）。

### 5.3 AI 设计 ＋ 平台规则适配
**做：** **人 + AI 对话窗口协作**（与建站智能体一致）——AI 在对话中给选题建议（取知识库＋市场），用户选题、并用大白话逐平台调整文案/语气/规格/发布时间，AI 实时改；图文一体生成（图＋文案＋标签）套品牌资料包，支持文生图、图生图、抠图换背景；**覆盖视频平台但不生成视频成片**，为 Reels/TikTok/Shorts/VK Клипы/RuTube 输出视频脚本、分镜建议、封面图、标题、文案、标签、规格校验、追踪链接和发布清单；**核心产出＝内容包**（同一选题多平台达标版本＋每份预埋追踪链接＋“发哪/配什么文案标签/什么时间发”清单）；一键导出。  
**覆盖平台（V1）：** LinkedIn · Facebook · Instagram · Reels（IG/FB）· TikTok · YouTube · Shorts · **VK Клипы** · **RuTube**。每平台按当前规则适配尺寸/比例/文案长度/标签/本地化：图文 1:1 / 4:5 / 1.91:1；视频平台输出 9:16 或 16:9 的脚本/分镜/封面/发布规格，不执行剪辑渲染；俄语市场用 VK/RuTube（西方平台在俄受限），文案本地化俄语、标签按俄语平台习惯。  
**GPT 最新规则校正：** 平台规则做成**可配置规则库 + GPT（OpenAI）按最新规则给建议并自动校正**——例如文案长度、标签、本地化、TikTok 前 150 字符前置钩子、视频平台时长/比例/封面规格提醒。V1.0 只做脚本与发布规格校验，不做成片剪裁、TTS 配音、自动字幕。规则库定期同步平台变更。  
**建站侧配套：** 落地页自动注入社媒分享卡元数据（Open Graph / VK），分享到各平台时预览卡正确渲染。  
**不做（→V1.5）：** 轻量短视频成片、TTS 配音、自动字幕。  
**不做（→V2）：** 复杂文生视频、数字人、平台一键发布（V1 手动发布＋追踪链接）。

### 5.4 询盘归集 → CRM
**做：** 两来源＝落地页表单 ＋ **专属系统邮箱**；进入即解析来源（UTM）标“来自 X 平台 Y 内容”；自动去重；自研最小 CRM（客户/联系人/商机阶段/跟进记录/来源字段）；拖拽式商机管道 ＋ 到期跟进提醒；界面极简、预留扩展字段。  
**邮件归集补充：** V1.0 需明确系统邮箱格式、租户映射、入站 webhook、邮件解析、附件存储、去重规则、垃圾过滤、审计日志；邮件正文与附件默认按潜在隐私处理。  
**不做（→V2）：** 社媒私信/评论归集、连客户 IMAP、复杂自动化工作流、重型 CRM。

### 5.5 AI 首响 / 打分 / 看板
**做：** 新询盘→AI 多语言首响草稿（**由本地 Qwen 生成**，取知识库，答基本问题＋引导下一步，保护客户隐私）＋**人工一键发送（HITL）**，草稿可编辑、可设模板；规则式 A/B/C 线索打分（可解释、规则可配置）；轻量看板（今日新询盘/待跟进/**来源分布：平台→内容→询盘数**）。  
**不做（→V2）：** 全自动回复、模型打分、完整 ROI（成本/收入）核算。

## 6. 多语言方案

- **语种：** 英语、阿拉伯语、俄语、法语、德语、葡萄牙语（6 种）。
- **流程：** OpenAI 出源语 → Google 翻译出其余语种（术语表锁定）→ 高营销度文案 LLM 直出或人工复核。
- **阿拉伯语：** 从右到左（RTL）布局，模板需 RTL 兼容，UI 镜像，单独 QA。
- **URL：** 每语种独立路径/后缀（如 `站点/ar`、`站点/ru`）＋ hreflang 标注，利于 SEO/GEO 与正确语言展示。

## 7. SEO ＋ GEO（V1 优化清单）

**SEO：** 自动生成 TDK、结构化标题、sitemap、hreflang；页面加载速度优化；图片 alt。  
**GEO（让网站被 AI 引擎引用）：**
- AI 爬虫可达——robots.txt 放行 AI bot；关键内容服务端渲染（不藏在 JS 后）；CDN 不拦 AI bot。
- JSON-LD 结构化数据。
- 页面顶部“问题式快答块”：用买家真实提问当小标题，下面纯段落给可核实事实。
- FAQ 区对齐真实买家提问。
- 事实密度：参数、认证等可核实事实**直接取自知识库**。
- 内容新鲜度：建议 7–14 天刷新节奏。

**GEO 可见度监测面板（各 AI 引擎提及率/引用率/位置）→ V1.5。**

---

## 8. 验收标准（“闭环跑通”判定）

**主标准（样本公司版）：** 用一家真实样本公司，平台从零导入知识库 → 生成响应式多语言落地页 → 生成一周 9 平台内容包（图文内容可用；视频平台提供脚本/分镜/封面/文案/标签）→ 手动发布（内容含预埋追踪链接）→ 至少产生 **1 条可追溯来源的询盘** → 自动进 CRM → AI 生成多语言首响草稿并由人工一键发出 → 看板能显示“该询盘来自 X 平台的 Y 内容 / 追踪链接”。  
**软指标：** 真实用户（一旦接入）愿意继续使用/付费。  
**关键判定点：** “可追溯来源的询盘”——证明追踪主线全程贯通。

---

## 9. V1.5 / V2 明确预留（V1 不做、架构留接口）

**V1.5：** 轻量短视频成片、TTS 配音、自动字幕、GEO 可见度监测面板。  
**V2：** 社媒/视频**自动发布连接器**、社媒私信/评论询盘归集、付费广告投放 API、**客户机房级真·私有化部署**（平台级本地 Qwen 已在 V1）、复杂文生视频与数字人、知识图谱与图片 OCR、自定义域名、拖拽全站编辑器、模型式线索打分、自动回复、完整 ROI 核算。

## 10. 待办与风险提示

1. **尽快找真实试点客户**——自跑有盲点，内容质量与获客效果需真实场景验证。
2. **阿拉伯语 RTL** ——额外开发与测试工作量，排期需预留。
3. **本地 Qwen 部署与迁移**——隐私路径已由本地 Qwen 解决（个资不出境）。开发/自测用个人电脑即可；**接入真实客户/上线前必须迁移到自有云 GPU 或服务器**（个人电脑无法支撑并发与 7×24）。置于标准推理端点后，迁移仅改配置。GPU 与运维成本随阶段递增。
4. **追踪链接服务**——闭环根基，建议最早开发、最先联调（建站表单 ↔ 设计内容 ↔ CRM ↔ 看板）。
5. **OpenAI/Google 成本与限流**——按量计费，需做缓存、配额与降级预案。
6. **模型路由正确性**——务必保证含客户隐私的请求绝不误走 OpenAI/Google；路由规则需有 DataClassifier、兜底与审计日志。
7. **V1.0 范围纪律**——V1.0 做完整主闭环，但不做短视频成片/TTS/自动字幕；复杂代运营后台不阻塞 M0–M6。

## 11. 控制台交互约定

- **卡片可下钻**：工作台的闭环节点、统计数字、智能体卡片、通知项均可点击，直接进入对应模块详情（如点“询盘 18”→ CRM；点“内容待发”→ 设计/发布）。
- **通知中心（铃铛）**：顶栏铃铛汇总待处理事项——新询盘待跟进、首响草稿待审批、内容包待发布、网站改动待上线等，点击直达。是 HITL 审批流的统一入口。
- **HITL 审批流**：凡对外动作（发布内容、网站上线、回复客户）默认进“待你确认”队列，可在通知中心或各模块一键确认/调整。
- **UI 数据契约**：高保真 HTML 只作为 UI 参考，正式实现需拆成 Next.js 组件，并补 `/docs/ui-data-contract.md`，明确每个页面的 API 与字段。

## 12. 平台底座：架构 · 多租户 · 权限 · 数据隔离 · 登录

> 这一层是 SaaS 的地基，需与业务功能并行设计。**基础隔离与登录从第一天就要做对；复杂多租户代运营后台放 M7，不影响 M0–M6 闭环验证。**

**12.1 系统架构（分层）**  
应用层（Web 控制台）→ API 网关（鉴权/限流/审计）→ 业务服务（建站/知识库/设计/CRM/首响）→ 模型网关（OpenAI/Google/本地 Qwen 路由）→ 数据层（关系库 + 向量库 + 对象存储）。模型网关与渲染、发布等耗时任务走异步任务队列。

**12.2 多租户与数据隔离（重点）**
- 每个企业客户＝一个租户（tenant）。所有业务数据带 `tenant_id`，查询强制租户过滤，杜绝越权读取。
- **向量库与对象存储按租户隔离**（独立 namespace / bucket 前缀），知识库与素材不串户。
- 敏感分级：标“仅内部”的知识与全部客户个资只在本地 Qwen 侧处理，不出境（见第 4 节）。
- V1.0 做基础租户隔离；复杂服务商代运营跨租户切换、套餐/用量、平台运营后台统一进 M7。
- 企业级可选**物理隔离**（独立库/独立部署）→ 与客户机房私有化一并列 V2。

**12.3 账户与登录**
- 邮箱 + 密码登录，强制 **2FA（二次验证）**；密码加盐哈希存储。
- 会话用短时令牌 + 刷新令牌；关键操作（发布、删库、改权限）二次确认。
- 企业版预留 **SSO / 企业微信 / 飞书**登录（V2）。

**12.4 权限管理（RBAC）**
- V1.0 角色：**Owner（企业主）/ Admin / 运营 / 业务员 / 只读**。
- 按模块 × 操作授权（如业务员只能看分配给自己的线索、不能改模型配置；运营能发内容但发布需 Owner/Admin 审批可配）。
- 服务商/代运营场景：一个账号管理多个客户租户，需“跨租户切换 + 操作留痕”，该能力进入 M7，不阻塞 V1.0 主闭环。

**12.5 平台管理（运营后台）**
- V1.0 客户侧必须有：成员与权限、品牌资料包、知识库、模型偏好、用量查看。
- 我方平台运营侧：租户管理、套餐与用量（Credits）、模型路由与配额、内容审核与合规、系统监控与告警 → M7。

**12.6 安全与合规基线**
- 传输 TLS、敏感字段加密存储；全量操作审计日志（谁在何时对哪条数据做了什么）。
- 数据出境最小化（个资走本地 Qwen）；GDPR/PIPL 的数据导出与删除能力预留。
- 第三方密钥（OpenAI/Google）集中保管、按租户配额、异常熔断。
