# TradePilot AI · V1.0 UI 数据契约（M0）

> **作者：** Claude（架构）· **状态：** 待签字冻结 → 交 Codex 实现（前端绑定）· **配套：** 《API 契约》《Schema 契约》
> 目的：每个页面的数字/列表/按钮都对应明确的 API 字段与状态，杜绝"页面做完接不上真实链路"。

---

## 0. 全局约定

- **取数**：页面只通过《API 契约》端点取数，不直连数据库。
- **状态机（共用）**：`loading → ready | empty | error`；异步任务额外有 `processing(轮询 jobs)`。
- **空态**：每个列表/看板都要有"还没有 X，去做 Y"的空态（带下一步按钮），不留白屏。
- **失败态**：模型失败 / 翻译失败 / **本地 Qwen 不可用(503)** / 网络错误，给可重试提示；隐私任务失败明确告知"已排队等本地引擎恢复，未发往第三方"。
- **RBAC 联动**：按 `GET /api/me` 的 role 控制按钮可见性（viewer 不显示发布/发送；sales 只见自己线索）。
- **移动端审批**：控制台桌面优先，但 `<HitlAction>` 的上线 / 发布 / 发首响审批按钮在移动端也必须可见、可点（核心审批不限设备）。
- **HITL 统一组件 `<HitlAction>`**：网站上线 / 内容发布 / 首响发送共用一套——状态：`待确认 → 已提交审批 → 已通过/已拒绝`；未通过不显示"已生效"。
- **长任务反馈 `<JobProgress jobId>`**：轮询 `GET /api/jobs/:id`，展示阶段（解析/切块/向量化/生成中）+ 进度条。

## 1. 登录 / 2FA
- 取数：`/api/auth/*`、`/api/me`。
- 绑定：登录表单 → login；若返回 `2fa_required` 显示验证码输入 → 2fa/verify。
- 状态：未登录 / 待 2FA / 已登录（带 currentTenant 与 role）。
- 多租户：`me.memberships` 多于 1 时显示租户切换器（写回 `X-Tenant-Id`）。

## 2. 工作台 Dashboard
- 取数：`GET /api/dashboard/summary?range=`、`GET /api/notifications`。
- 绑定：闭环图节点 / 三个统计数字（本周询盘=inquiriesCount、内容待发=pendingPublish、首响中位耗时）/ 来源分布 = `sourceAttribution[]`。
- **可下钻**：节点与数字点击 → 路由到对应模块（询盘→/crm；内容→/design；知识库→/kb…）。
- "待你确认"卡 = `pendingHitl`（按 type 分组）；点击进 HITL。
- 空态：无数据时显示"从导入知识库开始"引导按钮（向导入口）。

## 3. 知识库 Knowledge Base
- 取数：`/api/kb/documents`、`/api/kb/reviews`、轮询 `/api/jobs/:id`。
- 绑定：上传区 → POST documents；处理列表每行状态 = document.status（uploaded/parsing/chunking/embedding/ready/failed）+ `<JobProgress>`；敏感级标签 = sensitivity(public/internal_only)。
- 审核卡片 = reviews（question/answer/source_citation）；按钮 → 核准/修正/标敏感级。
- 概况 = 文档数/卡片数/已核准/语言数/仅内部数。
- 空态：无文档 → "把产品手册、报价单拖进来"。失败态：解析失败可重试。
- 用户提示：上传时给"哪些算可公开 / 仅内部"的默认建议（用户视角必补项）。

## 4. AI 建站 Site Builder
- 取数：`/api/sites/:id`、`/api/sites/:id/chat`、`/api/jobs/:id`。
- 绑定：左聊天 = chat 往返；右预览 = site_locales.translated_content；语言标签 = locales（en/ar/ru/fr/de/pt），**ar 切换 direction=rtl**；URL = `url_path`（带语种后缀）。
- 状态徽章：SEO ✓ / GEO ✓ / 响应式 / OG·VK 分享卡 / 追踪链接（来自 version 的 seo/geo/og_meta）。
- 发布 = `<HitlAction type=site_publish>`（publish-request → 审批后上线）。
- **预览体检**（用户视角必补）：展示 SEO/GEO/移动端/询盘表单的检查结果清单。

## 5. 站点管理 Site Management
- 取数：`/api/sites`。
- 绑定：站点列表（名称/URL/版本/语言）；上下线 toggle → status；版本回滚 → rollback。
- **AI 自动补全面板** = `/api/sites/:id/autofill` 候选（新产品/认证/博客草稿）；每条"调整 / 确认上线（HITL）"。

## 6. AI 设计 / 内容包 Content Pack
- 取数：`/api/content-packs/:id`、`/api/content-packs/:id/chat`、`/api/platform-rules`、`/api/jobs/:id`。
- 绑定：左"人+AI 对话" = chat（选题建议 + 逐平台调整）；右 9 平台卡 = content_items（platform/media_type/spec/body/trackingLink）。
- **视频平台卡**：media_type=`video_script` → 显示脚本/分镜/封面，**不显示成片播放**（成片→V1.5）。
- GPT 校正面板 = chat 返回的规则校正项。
- 发布清单 = items（平台/规格/计划时间/追踪链接/标记已发）；"标记已发" → mark-published（绑 content_item_id）；导出 → export。
- 优先级建议（用户视角必补）：AI 标出"先发哪条"。

## 7. 发布清单 Publish Checklist（可并入内容包页）
- 字段（运营视角必补）：内容负责人 ownerUserId / 计划发布时间 plannedAt / 实际发布时间 publishedAt / 平台 platform / 发布状态 publishStatus(待发/已发/失败/已下线) / 追踪链接状态。
- 操作：标记已发 / 撤回 / 过期提示。

## 8. 询盘线索池 Lead Pool
- 取数：`/api/crm/leads`。
- 绑定：每行 = lead（公司/国旗=country/评分=score A·B·C/来源=platform→content/语言/状态/首响状态）。
- **score 解释**（用户视角必补）：点开看 score_reason"为什么是 A 级"。
- 空态：无线索 → "去生成内容包获客"。

## 9. CRM 管道 CRM Pipeline
- 取数：`/api/crm/opportunities`。
- 绑定：看板列 = stage(new/contacted/quoted/won/lost)；卡片拖拽 → PATCH stage；到期提醒 = followUpDueAt。

## 10. AI 首响审批 Reply HITL
- 取数：`/api/replies/:id`、`/api/hitl`。
- 绑定：左客户询盘 inquiry.body（高亮关键诉求）；右 AI 草稿 reply.draftText（可编辑）+ 引用 citations + **"本地 Qwen·隐私不出门"徽章**。
- 发送 = `<HitlAction type=reply_send>`（approve 后才发出）。
- 失败态：本地引擎不可用 → "已排队，等恢复后生成，未发往第三方"。

## 11. 通知中心 Notification Center
- 取数：`/api/notifications`。
- 绑定：待处理询盘 / 待审批（内容/网站/首响）；点击直达对应模块；标记已读。是 HITL 统一入口之一。

## 12. 设置 Settings
- 成员与权限（members + role，RBAC）/ 品牌资料 brand_kits / 模型偏好 / 用量账单（usage：creditsBalance + 明细）。
