# Codex 指令前缀（每次粘在指令开头）

> 用法：把下面「通用约束」整段复制到你给 codex 的指令最前面，再接你这次具体要它做的事。
> 目的：让 codex 在这个仓库里安全工作，不改乱现有架构。

---

## 通用约束（复制这段）

```
你在 TradePilot AI 仓库（Next.js App Router + TS + Prisma/PostgreSQL，monorepo，业务在 apps/web）工作。
请严格遵守：
1. 优先只新增文件；如必须修改现有文件，先明确告诉我改哪些、为什么，等我确认。
2. 遵守 AGENTS.md 铁律：数据访问走 server/db/tenant-prisma.ts 并带 tenantId；
   模型调用走 server/model-gateway；客户隐私数据（姓名/电话/询盘正文）只走本地 Qwen，绝不外发。
3. 不接任何第三方外部服务（Gmail/WhatsApp/Stripe/Hunter/Langfuse 等已全部砍掉，别引入）。
4. 价格/交期/认证不得 AI 自动承诺；任何对外发送必须经现有 HITL；合规结论必须标"需专业机构确认"。
5. UI 复用 app/globals.css 组件类与 app-shell，不套整页背景、不硬编码颜色、不动现有数据获取与权限逻辑。
6. 一次只做一个阶段/一件事，不要顺手重构无关代码。
7. 完成后运行 npm run lint && npm run typecheck，并输出：新增文件清单、修改文件清单（如有）、剩余阻塞点。
```

---

## 用法示例

执行某个阶段任务：
```bash
codex "$(cat docs/codex_tasks/_指令前缀.md 的「通用约束」段) 然后执行 docs/codex_tasks/G4_compliance_local.md。"
```

更实用的写法（手动拼）：
```bash
codex "【通用约束：只新增文件不改现有；隐私走本地Qwen；不接第三方；合规标需专业机构确认；
完成后跑 lint+typecheck 并列文件清单】 请执行：在 server/ 新增 xxx skill，..."
```

让 codex review 我已写好的代码（不改动）：
```bash
codex "只读不改：review apps/web/server/quotation 这几个新文件，指出潜在 bug、
租户隔离漏洞、与 AGENTS.md 铁律不符之处，列出问题清单即可，不要改代码。"
```

做"需批准"的现有文件增强（明确告知会改现有文件）：
```bash
codex "【通用约束同上，但本次允许修改 server/replies/service.ts】
为 first_response_draft 增加 questions_to_confirm / uncertain_points / next_action，
存入 Reply.citations.meta（不改 schema、不改 draftText 纯文本）。完成后跑 lint+typecheck。"
```

## 提醒

- G1–G5 的本地代码已经写好并验证（typecheck/eslint 干净），通常**不需要**让 codex 重做这些阶段。
- codex 现在更适合：① review 现有代码；② 做"需批准"的两项现有文件增强；③ 你后续的新需求。
- 给 codex 的指令越具体越好：说清"改哪个文件、产出什么、不许碰什么、怎么验证"。
