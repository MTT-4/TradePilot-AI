# TradePilot AI · 风险应对行动建议（安全 + IP + 商业模式合一）

> 配套：`安全与风险审查.md`、`知识产权与商业模式风险.md`。本文是"怎么做"的行动版。

## 总原则（一句话）

高风险全在 **G3 及以后**（第三方数据、自动外发、支付）。最优风控不是去加固它们，
而是先把 **G1–G2 做成"全本地、不外发、不碰第三方数据"的合规可演示 MVP**，
之后每个外部连接器都当"准入闸"逐个开，过不了闸就不接。这样风险随业务可控释放。

## 三、外部连接器"准入闸"清单（当前已不适用：13 个连接器全部砍掉）

> 现状：全部 13 个第三方连接器已移出路线图，本阶段不接任何外部服务。
> 以下清单**仅在未来确需重新引入某个连接器时**才适用，届时逐条过闸。

- [ ] ToS 明确允许本用法（非转售、非群发、可商用）？
- [ ] 有 mock fallback，断供/封号不致系统崩溃？
- [ ] API Key 走环境变量 + 每租户隔离，不进日志？
- [ ] 出站经"出口闸"，PII 脱敏 + 同意校验？
- [ ] 监控类（Langfuse 等）trace 不落客户 PII（优先自托管）？
- [ ] 有审计留痕（谁发了什么、谁导出了什么）？

## 四、明确不做（do-not list）

- ❌ 转售/再分发第三方数据（Hunter/Apollo/ImportYeti）给你的客户。
- ❌ 无同意的自动群发开发信。
- ❌ AI 自动承诺价格/交期/认证，或自动发送、自动扣款。
- ❌ 默认拿租户数据去微调/训练模型。
- ❌ 把含 PII 的 prompt 发往第三方监控/模型。
- ❌ 引入 AGPL/GPL 传染性许可库进核心。

## 五、差异化（把合规变成卖点）

不要靠"连接器多"竞争——那易被复制。护城河押在：
**本地隐私优先路由（合规即卖点）+ 外贸行业 know-how + 租户数据闭环 + HITL 工作流**。
model-gateway 可插拔换模型，这点要守住，作为商业韧性与议价保护。

## 六、需要你拍板的业务决策（影响技术取舍）

- **首发市场**：欧盟优先 → GDPR/同意管理要最先做；国内优先 → PIPL；决定合规模块顺序。
- **对 AI 输出的责任边界**：纯辅助工具 vs 承诺质量——决定限责条款与保险需求。

> 已决策：数据获客（Hunter/Apollo 等）与第三方监控（Langfuse 等）均已**砍掉**，
> 故"build vs buy""自托管 vs SaaS"两项无需再决策。计费先用本地用量计量，不接 Stripe。
