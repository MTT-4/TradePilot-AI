# 发布前手工验收清单

## 1. 环境准备

- `nvm use`
- `docker compose up -d`
- `npm run prisma:migrate`
- `npm run prisma:seed`
- `npm run lint`
- `npm run typecheck`

## 2. 演示账号

- 租户 A Owner：`owner-a@tradepilot.local` / `TradePilot@2026`
- 当前登录策略：未开启 2FA 的账号可直接登录；已开启 2FA 的账号仍需验证码。

## 3. 闭环主线验收

1. 登录 `/login`
- 用演示账号登录成功，跳转到 `/`
- 退出登录后重新访问后台页会回到 `/login`

2. 工作台 `/`
- 工作台卡片正常加载
- “待你确认”可跳 `/replies`、`/sites`、`/design`
- 来源归因区能跳 `/crm`

3. 知识库 `/kb/reviews`
- 上传文件后能看到文档与任务状态
- 可修改审核状态并标记“已核准”
- `INTERNAL_ONLY` 文档不会进入公开营销生成链路

4. AI 建站 `/sites`
- 可创建站点草稿
- 站点详情能看到多语种 URL、预览体检、版本历史
- 发布动作会进入 HITL，未审批前不会直接上线

5. AI 设计 `/design`
- 可创建内容包
- 内容包对话页可继续编辑各平台内容
- 生成后的内容项能进入发布清单与追踪链路

6. 发布清单 `/publish-checklist`
- 能看到待发布内容
- 计划时间、发布状态、追踪链接展示正常
- 导出入口可用

7. 追踪链接 `/tracking-links`
- 列表能显示 slug、UTM、点击量、关联线索
- `owner/admin/operator` 可编辑目标地址、UTM、Bot 过滤
- 保存后详情卡里的“服务端最终跳转”会同步变化
- 访问 `/t/:slug` 时，最终跳转始终以服务端 UTM 为准

8. 询盘与 CRM `/crm`
- 表单询盘与邮件询盘都能进入线索池
- 线索详情能看到询盘、活动、来源归因
- 商机阶段可修改

9. AI 首响 `/replies`
- 能从 CRM/询盘入口起草首响
- 左侧询盘正文、右侧草稿可正常展示
- 可编辑、拒绝、确认发送
- 客户隐私内容只走本地 Qwen

10. 运维与治理
- `/jobs` 能查看最近任务及详情
- `/notifications` 能查看通知并执行已读/忽略
- `/hitl` 审批记录完整
- `/settings` 可查看成员、品牌、模型策略、数据请求、审计摘要

## 4. 多租户与权限

1. 多租户隔离
- 切换租户后，站点/内容包/CRM/追踪链接数据全部随租户变化
- A 租户内容在 B 租户不可见

2. 权限控制
- `viewer` 只能查看，不能看到编辑/发送/发布能力
- `sales` 可处理 CRM 与首响，但不能改站点/内容发布设置
- `owner/admin/operator` 可编辑追踪链接、内容、站点流程

## 5. 发布阻断项

任一项失败，本次不发布：

- `npm run lint` 或 `npm run typecheck` 未通过
- 追踪链接点击后服务端归因不一致
- 询盘无法追溯到 tracking link / platform / content
- 首响草稿走了第三方模型
- HITL 未审批却能直接对外发送或上线
- 跨租户能看到不属于当前租户的数据
