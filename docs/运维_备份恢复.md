# 运维备份与恢复

## 目标

- 备份 PostgreSQL 数据库
- 备份对象存储桶内容
- 在本地或演练环境执行恢复
- 在进入真实客户数据前，先做一次恢复演练

## 依赖

- `pg_dump`
- `pg_restore`
- `psql`
- `mc`（MinIO Client）
- 可用的 `.env`

## 备份

```bash
npm run ops:backup-local
```

默认输出到 `backups/<timestamp>/`，包含：

- `db/tradepilot.dump`
- `db/schema.sql`
- `object-store/<bucket>/`
- `manifest.json`

## 恢复

```bash
npm run ops:restore-local -- ./backups/<timestamp> --yes
```

说明：

- 该命令会覆盖目标数据库中的现有对象
- 恢复前会确保 `vector` 扩展存在
- 如果备份目录里带对象存储快照，会同步回桶

## 密钥检查

```bash
npm run ops:check-secrets
```

会阻止以下情况：

- `AUTH_SECRET` 仍是占位值
- `OPENAI_API_KEY` 仍是占位值
- `GOOGLE_TRANSLATE_KEY` 仍是占位值
- `S3_SECRET_KEY` 仍是占位值
